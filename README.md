Описание функциональности
Проект представляет собой консольное приложение для управления базами данных студентов, работающее полностью в оперативной памяти. Поддерживается работа с несколькими таблицами.

Реализованные возможности:
Создание записи (Create)
· Добавление нового студента в выбранную таблицу с полями: ID, имя, фамилия, возраст, пол
· Проверка уникальности ID в пределах таблицы
· Валидация возраста (не может быть отрицательным)

Чтение записей (Read)
· Просмотр всех студентов в выбранной таблице
· Поиск студентов по фильтрам:
· ID
· Имя
· Фамилия
· Возраст
· Пол

Обновление записей (Update)
· Изменение данных существующего студента по ID в выбранной таблице
· Возможность обновить только выбранные поля

Удаление записей (Delete)
· Удаление студента по ID из выбранной таблицы
· Подтверждение операции перед удалением

Управление таблицами
· Создание новой таблицы с указанием названия
· Просмотр списка всех существующих таблиц
· Выбор активной таблицы для выполнения операций

Типы данных:
· Запись студента представлена словарем с ключами: id, first_name, second_name, age, sex
· Данные хранятся в глобальном словаре tables в модуле memory.py, где ключ — название таблицы, значение — список студентов

Тестирование приложения
Запуск тестов
Для проверки работоспособности приложения используется модуль unittest. Тесты расположены в директории tests/:

Запуск всех тестов выполняется командой из корневой папки проекта:

bash
python -m unittest discover tests
Для запуска конкретного теста:

bash
python -m unittest tests.test_memory
python -m unittest tests.test_tui
Для более подробного вывода используйте флаг -v:

bash
python -m unittest discover tests -v
Покрытие тестами
test_memory.py — тестирование backend-логики
Проверяются следующие сценарии работы с базой данных:

Управление таблицами
· Создание новой таблицы
· Попытка создания таблицы с существующим названием (должна вызывать исключение)
· Получение списка всех таблиц

Создание записи (Create)
· Успешное добавление нового студента в указанную таблицу
· Попытка добавления с существующим ID в таблице (должна вызывать исключение)
· Попытка добавления с отрицательным возрастом (должна вызывать исключение)

Чтение записей (Read)
· Получение списка всех студентов из таблицы
· Поиск студента по существующему ID в таблице
· Поиск студента по несуществующему ID (должен возвращать None)
· Поиск студентов по фильтрам (имя, фамилия, возраст, пол)

Обновление записей (Update)
· Успешное обновление существующего студента в таблице
· Попытка обновления несуществующего студента (должна вызывать исключение)
· Частичное обновление (только выбранные поля)

Удаление записей (Delete)
· Успешное удаление существующего студента из таблицы
· Попытка удаления несуществующего студента (должна вызывать исключение)

test_tui.py — тестирование пользовательского интерфейса
Проверяются следующие сценарии работы с TUI:

Отображение меню
· Корректный вывод главного меню с 8 пунктами (0-7)
· Обработка ввода пользователя

Валидация ввода
· Проверка корректности вводимых данных (возраст, пол, ID)
· Обработка пустых значений
· Проверка названий таблиц

Интеграция с backend
· Вызов соответствующих функций БД при выборе пунктов меню
· Корректное отображение результатов операций с таблицами
